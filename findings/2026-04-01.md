Suggested Improvements
----------------------

### Skills

**Apply the "distributional convergence" pattern to your existing skills**
- **What:** Review skills like `frontend-design`, `ux-designer`, and any domain-specific skills for explicit anti-default guidance
- **Why:** The [frontend design skills post](https://claude.com/blog/improving-frontend-design-through-skills) demonstrates that Claude converges on generic outputs unless skills explicitly name what to avoid. This applies beyond frontend — any skill where Claude tends toward "safe" defaults benefits from negative examples.
- **How:** For each skill, add a short "avoid these defaults" section naming the specific generic patterns Claude falls into for that domain. Keep it under 100 tokens per skill — the blog's example skill was only ~400 tokens total.
- **Impact:** Medium
- **Confidence:** Likely

**Create a skill-authoring meta-skill informed by Anthropic's "prompt altitude" concept**
- **What:** Update `/claude-skill-management` or `/prompt-engineer` to reference the "prompt altitude" principle from the blog — guidance should be neither overly prescriptive nor vaguely aspirational
- **Why:** The blog post names this as the key design principle for effective skills. Your existing skill-creation workflow could benefit from this as an explicit quality check.
- **How:** Add a checkpoint in the skill creation process: "Is this skill at the right altitude? Too prescriptive = brittle. Too vague = ignored."
- **Impact:** Medium
- **Confidence:** Likely

### AgenC Configuration

**Evaluate Claude Code on the web as a complement to AgenC for parallel task execution**
- **What:** Investigate whether Claude Code's web interface (claude.com/code) could handle some parallel workloads currently managed through AgenC missions
- **Why:** The [web Claude Code post](https://claude.com/blog/claude-code-on-the-web) describes parallel task execution across repos on managed infrastructure — similar to what AgenC missions provide but without local resource consumption.
- **How:** Try running a few typical AgenC-style tasks (e.g., cross-repo investigations, parallel bug fixes) through the web interface. Compare the workflow friction and capability against spawning AgenC missions.
- **Impact:** Medium
- **Confidence:** Speculative — depends heavily on whether the web interface supports the same level of customization (skills, hooks, MCP servers) that AgenC provides

### Workflows

**Adopt the screenshot-based debugging pattern from Anthropic's internal teams**
- **What:** Formalize feeding screenshots (dashboards, error UIs, browser state) into Claude Code debugging sessions
- **Why:** The [internal usage post](https://claude.com/blog/how-anthropic-teams-use-claude-code) describes the Data Infrastructure team diagnosing a Kubernetes pod scheduling failure by sharing "dashboard screenshots" — Claude Code is multimodal and can read images, but this isn't explicitly part of your debugging workflow.
- **How:** Update `/systematic-debugging` or create a lightweight convention: when debugging involves a UI or dashboard, screenshot the relevant state and feed it to Claude Code rather than describing it textually.
- **Impact:** Low
- **Confidence:** Likely

**Consider MCP-based documentation consolidation**
- **What:** Use MCP servers to connect Claude Code to scattered documentation sources (Notion, Google Docs, Confluence, etc.)
- **Why:** Anthropic's internal teams report ~80% reduction in research time by consolidating docs through MCP into markdown runbooks. You already use MCP (Todoist) so the infrastructure pattern is familiar.
- **How:** Identify your most-referenced documentation sources. Check the [integrations post](https://claude.com/blog/integrations) for available MCP connectors (Confluence, Jira, Linear, etc.). Set up relevant ones.
- **Impact:** Medium
- **Confidence:** Likely — depends on which documentation tools you use

### Claude Code Settings

**Explore the Claude for Chrome extension for build-verify-debug loops**
- **What:** Install the Chrome extension and integrate it with Claude Code workflows
- **Why:** The [Chrome post](https://claude.com/blog/claude-for-chrome) specifically highlights integration with Claude Code: "build in the terminal, verify in the browser, and debug with Claude reading console errors and DOM state directly." For frontend work, this closes the feedback loop between Claude Code changes and browser verification.
- **How:** Install the extension, grant it permissions for development domains (localhost, staging). Test with a frontend task to evaluate whether the integrated workflow is faster than manual browser verification.
- **Impact:** Medium (for frontend work), Low (if most work is backend)
- **Confidence:** Likely
